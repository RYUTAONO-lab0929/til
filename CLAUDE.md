# CLAUDE.md - TILリポジトリ運用ガイド

## このリポジトリについて
日々の技術的な学びを記録するTIL（Today I Learned）リポジトリ。
記録は将来、職務経歴書の自動生成やスキルの棚卸しに活用する。

## ディレクトリ構成
```
til/
├── README.md              # カテゴリ一覧と記事数（自動更新対象）
├── categories/{category}/  # カテゴリ別のTIL記事
│   └── YYYY-MM-DD-slug.md # 個別記事
└── templates/
    └── til-template.md    # 記事テンプレート
```

## TIL記事作成の指示

### 基本コマンド
ユーザーが「今日のTIL書いて」「TIL: 〇〇について」などと言ったら以下を行う：

1. `templates/til-template.md` を元に記事を作成する
2. ファイル名は `categories/{category}/YYYY-MM-DD-{slug}.md` とする
   - slug は英語のケバブケース（例: `array-destructuring`）
   - 日付は今日の日付を使う
3. ユーザーの説明を元に各セクションを埋める
   - ユーザーの説明が少ない場合は質問して補完する
   - 「背景・きっかけ」と「業務インパクト」は特に重要。必ず聞き出す
4. 記事作成後、README.md のカテゴリ一覧の記事数を更新する
5. 新しいカテゴリの場合は README.md にカテゴリ行を追加する

### 記事の品質基準
- 技術用語は正確に書く
- コード例はできるだけ含める
- 会社名・プロジェクト名・内部APIなどの機密情報は絶対に含めない
- 一般的な技術課題として抽象化して書く

### commit & push
- コミットメッセージ: `til: {カテゴリ} - {タイトル}`
  - 例: `til: typescript - Mapped Typesでオブジェクトの型を変換する`
- 記事作成とREADME更新は1つのコミットにまとめる
- 作成後に自動で `git push origin main` する

## 職務経歴書生成の指示

ユーザーが「職務経歴書作って」「レジュメ生成して」などと言ったら：

1. `categories/` 配下の全記事を読み込む
2. 以下の観点で分析する：
   - 使用技術の頻度と深さ
   - 課題解決のパターン（パフォーマンス改善、設計改善、DX向上など）
   - 業務インパクトの実績
   - 技術領域の幅と専門性
3. 分析結果を元に職務経歴書のドラフトを生成する
4. 技術スタック、強み、実績を具体的な数字付きでまとめる

## 振り返りの指示

ユーザーが「今月の振り返り」「最近何学んだっけ」などと言ったら：

1. 指定期間（デフォルト: 直近30日）の記事を集計する
2. カテゴリ別の記事数、主要な学び、成長ポイントをまとめる
3. 学びの傾向やギャップ（学んでいない領域）を指摘する

## 言語
- ユーザーとの会話、記事の本文: 日本語
- ファイル名、slug、コミットメッセージのカテゴリ部分: 英語
